Namespace Microsoft.UpdateServices.Metadata

Classes

Classification

Represents a Classification. Used to clasify updates on an upstream server.

Example classifications: drivers, security updates, feature packs etc.

Detectoid

Represents a detectoid. Detectoids determine applicabilty of updates for a computer and as such are used as pre-requisites for other updates.

Example detectoids: x64, x86, arm64, DirectX12 supported, etc.

DriverFeatureScore

Stores driver feature score.

For more information, see driver feature score documentation

DriverMetadata

Extended information about a hardware ID handled by a driver update. A driver update matches one or more hardware ids; each match has its own metadata that describes it.

DriverUpdate

Represents a driver update.

DriverVersion

Stores driver versioning data, comprised of driver date and 4 part version. Implements custom comparison and equality check for driver version

Identity

Represents the identity of an update.

An update's identity is the pair ID (Guid) - Revision (integer).

MetadataFilter

Represents a filter that can be applied to update metadata

Product

Metadata for a product category.

QueryFilter

Represents a filter used for quering updates. Combines categories and classifications filters

To create a QueryFilter, query the categories on the upstream server first.

SoftwareUpdate

Reprerents a software update.

Update

A base class for all updates stored on an upstream update server. Stores generic update metadata applicable to both categories and updates.
